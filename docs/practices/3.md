# 包管理工具

前端包管理工具用于安装、版本管理、依赖解析和发布第三方模块。随着生态发展，出现了多种工具：`npm`、`yarn`、`pnpm`、`cnpm`，以及构建层面的 `npx`、`pnpx`。它们在依赖安装策略、磁盘占用、性能、monorepo 支持等方面各有差异。常见对比维度包括：是否支持 workspace（monorepo）、是否使用硬链接/符号链接复用磁盘空间、lockfile 的形式、以及是否兼容 npm registry。选型时通常在性能、磁盘占用、生态兼容性和团队规范之间权衡。

## npm 是什么？

npm是Node.js的包管理工具，它的诞生也极大的促进了前端的发展，在现代前端开发中都离不开npm的身影。

常见的使用场景有以下几种：

允许用户从NPM服务器下载别人编写的第三方包到本地使用。
允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

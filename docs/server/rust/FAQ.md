# FAQ?

## Rust 中 move、copy、clone 三者的区别是什么？分别在什么时候发生？

| 行为    | 是否自动 | 是否分配堆内存  | 适用于哪些类型                  |
| ------- | -------- | --------------- | ------------------------------- |
| `move`  | ✅        | ❌（但转移指针） | 默认所有权转移类型（如 String） |
| `copy`  | ✅        | ❌               | 实现了 `Copy` 的简单类型        |
| `clone` | ❌        | ✅               | 所有实现 `Clone` 的类型         |

### ✅ 安全 + 性能 + 显式控制

- **安全**：防止双重释放（move 会使原变量无效）
- **性能**：Copy 避免不必要的堆操作，轻量快速
- **控制力**：Clone 让开发者自己决定是否需要深拷贝，避免隐藏的性能陷阱

# API 设计

前后端数据契约的设计与实现。聚焦「前端如何取数」这一应用层，区别于 [网络协议](../network/)（传输层）。

| 主题 | 说明 |
| --- | --- |
| [GraphQL](../graphql/) | 按需查询的 API 语言与运行时。 |
| [REST / API 设计 FAQ](./FAQ.md) | RESTful 规范、GET/POST 语义、幂等性等常见问题。 |

## 学习路径

1. 理解 REST 架构风格与 RESTful 规范。
2. 掌握 HTTP 方词语义（GET/POST/PUT/PATCH/DELETE）与幂等性。
3. 对比 REST 与 GraphQL，理解各自的场景与取舍。
4. 设计清晰的资源命名、版本化、分页与错误模型。

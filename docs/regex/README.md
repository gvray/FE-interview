# 正则表达式

正则表达式（Regular Expression，简称 RegExp）是前端面试中极具区分度的高频考点。无论是表单校验、字符串解析、模板替换，还是编辑器查找替换、URL 路由匹配，正则都扮演着不可替代的角色。掌握正则不仅能体现候选人对字符串处理的功底，更能在面对复杂业务时写出更简洁、更高效的代码。

在前端面试中，正则相关的考察通常不会停留在"背语法"层面，而是会给出具体场景让你手写正则：手机号校验、邮箱校验、千分位格式化、驼峰转换、敏感词过滤……这些题目看似简单，却能在短时间内暴露候选人是否真正理解贪婪匹配、断言、捕获组、回溯等核心概念。

## 章节概览

本章节按照由浅入深的顺序，系统梳理正则在前端面试中需要掌握的全部知识点：

- [正则基础语法](./syntax-basics.md)：字面量与 RegExp 构造函数、字符与字符类、锚点、修饰符。
- [元字符与量词](./meta-and-quantifiers.md)：元字符语义、量词与贪婪/懒惰/占有模式、分组与反向引用、或与转义。
- [断言与回溯](./assertions.md)：先行/后行断言、回溯原理、灾难性回溯与原子组。
- [常用正则示例](./common-patterns.md)：邮箱、手机号、身份证、URL、IP、日期、千分位、密码强度等实战正则。
- [进阶用法](./advanced.md)：String 方法与正则的配合、replace 回调与 `$1`、命名捕获组、RegExp 属性、性能与可读性权衡。

## 学习路径

建议按以下顺序学习：

1. **先打基础**：通读 [syntax-basics.md](./syntax-basics.md) 与 [meta-and-quantifiers.md](./meta-and-quantifiers.md)，理解字符类、量词、分组这些"词汇"。
2. **再攻难点**：[assertions.md](./assertions.md) 是正则的分水岭，断言与回溯是区分"会用正则"与"精通正则"的关键。
3. **实战巩固**：[common-patterns.md](./common-patterns.md) 中的每个正则都建议自己手写一遍，对照解析查漏补缺。
4. **融会贯通**：[advanced.md](./advanced.md) 将正则与 String 方法、性能优化结合，覆盖面试中"会用正则"之外的工程化考察。

> 一句话：正则是前端工程师的"内功"之一，写得出一行精准的正则，胜过十行冗长的字符串切片代码。
